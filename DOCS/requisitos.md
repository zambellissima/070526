# Levantamento de Requisitos

## Visão Geral

O sistema tem como objetivo auxiliar na gestão de uma rede de cinemas, permitindo o controle de filmes em cartaz, sessões, cinemas e público das exibições.

O sistema será utilizado por funcionários/administradores da rede e também permitirá consultas realizadas pelos espectadores.

---

# Requisitos Funcionais

## RF01 – Cadastrar Cinema
O sistema deve permitir o cadastro de cinemas da rede.

## RF02 – Consultar Cinema
O sistema deve permitir consultar os cinemas cadastrados.

## RF03 – Cadastrar Filme
O sistema deve permitir cadastrar filmes em cartaz.

## RF04 – Consultar Filme
O sistema deve permitir consultar informações dos filmes.

## RF05 – Criar Sessão
O sistema deve permitir criar sessões para exibição dos filmes.

## RF06 – Consultar Sessões
O sistema deve permitir consultar sessões disponíveis.

## RF07 – Registrar Público
O sistema deve permitir registrar a quantidade de espectadores de uma sessão.

## RF08 – Consultar Público por Sessão
O sistema deve permitir consultar o público registrado em cada sessão.

## RF09 – Consultar Público por Filme
O sistema deve permitir consultar o total de público de um filme.

## RF10 – Consultar Público por Cinema
O sistema deve permitir consultar o total de público de um cinema.

---

# Requisitos Não Funcionais

## RNF01
O sistema deverá utilizar arquitetura MVC.

## RNF02
O sistema deverá utilizar camada Service e Repository.

## RNF03
O sistema deverá utilizar banco de dados SQLite.

## RNF04
O sistema deverá possuir interface simples e objetiva.

## RNF05
O sistema deverá garantir persistência dos dados cadastrados.
